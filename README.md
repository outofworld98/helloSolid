# SOLID 
좋은 벽돌(Clean Code)로 좋은 아키텍처를 정의하느 원칙, 2004년 마이클 페더스의 의해서 탄생
* 함수와 데이터 구조를 클래스(함수와 데이터를 결합한 집합)로 배치하는 방법
* 변경에 유연, 이해하기 쉽다, 많은 소프트웨어 시스템에 사용될 수 있는 컴포넌트의 기반이 된다.
